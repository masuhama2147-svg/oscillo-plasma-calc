# 研究者向け「解説トグル付き」UI 再設計ドキュメント

**目的**: 現 UI は計算式が LaTeX 生ソース (`$$ V_{pp} = V_{\max} - V_{\min} $$` 等) のまま表示され、かつ「この値から何が言えるか」のガイドが無い。研究者（特に学生）が **波形→式→考察** を一画面で追えるように、各物理量を **①綺麗な式 ②意味 ③グラフの見どころ ④結果からの考察** の 4 ブロック構造に再設計する。本ドキュメントは実装前の **議論用 仕様書**。

---

## 0. 現状の課題（スクリーンショットから確認）

| 症状 | 原因 | 影響 |
|---|---|---|
| Trace タブで `$$ V_{pp} = V_{\max} - V_{\min} $$` が文字のまま | MathJax v3 が動的挿入された DOM を再 typeset していない | 数式が読めない |
| 「根拠論文: 2013_CAP-13-1050」だけでは **読み方が分からない** | 解釈レイヤが欠落 | 学生には敷居が高い |
| 代入済み式が `1.184e+04\mathrm{V}` 等 コンピュータ寄り表記 | `fr"..."` でフォーマットしたまま出力 | 見づらい |
| グラフ（Waveform / P(t) / Lissajous / FFT）と式が **横並びで見比べられない** | タブが分かれている | 考察と数値が結びつかない |

実機のオシロ波形例（ユーザ提示）: 20 μs スケールで 3〜4 発のパルスが確認でき、電圧 ±5 kV クラス、電流ピークの位置が電圧ピーク直後に立っている（ストリーマ到達の典型）。こうした **「グラフでこう見えたら、式のこの項がこの値になる」** という読み筋をシステムに内蔵する。

## 1. 再設計コンセプト

### 1.1 すべての物理量を `Explanation` 構造体で統一表示

```
┌─────────────────────────────────────────────┐
│ 〔物理量名〕   例: ピーク間電圧 Vpp        │
│ 数値カード (大きく): 11,840 V               │
├──────────────────── 解説トグル (開閉) ─────┤
│  ① 式              :  Vpp = Vmax − Vmin    │  ← KaTeX / MathJax 実レンダリング
│  ② 意味            :  波形の最大振れ幅。   │
│                       放電の強度指標。      │
│  ③ グラフの見どころ :  Waveform タブで     │
│                       上下ピークに線を引く  │
│                       と直感的。正負が対称 │
│                       ⇔双極性放電。         │
│  ④ 結果からの考察  :  11.8 kV は Paschen   │
│                       破壊電圧を大幅に超   │
│                       えており、液中スト    │
│                       リーマ形成確定。     │
│                       PW 条件では本条件     │
│                       が最大 → CO₂ 電離    │
│                       効率ピーク候補。      │
│  ⑤ 根拠論文         :  2013_CAP-13-1050   │
└─────────────────────────────────────────────┘
```

### 1.2 4 ブロック構造の命名

- `formula`  : 綺麗な式（LaTeX→レンダリング済み）
- `meaning`  : 物理的意味（2〜4行、平易な日本語）
- `graph_hint` : グラフで何をどう見るか（「Waveform タブの青線最大〜最小差」等、具体的な見方）
- `interpretation` : 結果からの考察（典型値・オーダー・野村研 研究への含意）

### 1.3 式レンダリング問題の解決策

**方針 A（採用候補）**: **KaTeX auto-render** を使う（MathJax より速く、DOM 挿入後も `renderMathInElement(document.body)` で即時レンダ可能）。

**方針 B**: MathJax v3 を残しつつ `reactive.effect` の末尾で JS `MathJax.typesetPromise()` を呼ぶ。

**方針 C**: サーバ側で sympy → MathML または SVG に事前変換し、HTML に直接埋める（オフライン完結、CDN 不要）。

| 方針 | 利点 | 欠点 |
|---|---|---|
| A KaTeX | 軽量・同期的、`delimiters: [{left:"$$",right:"$$"}]` で既存ソースそのまま使える | CDN 依存 |
| B MathJax 再 typeset | 既に CDN 読み込み済み、追加コード最小 | 動的挿入の度に JS 呼び出しが必要、タイミング問題 |
| C sympy→MathML | オフラインで堅牢、再現性最高 | sympy の MathML 出力は表示が素朴、微調整必要 |

→ **A (KaTeX)** を第一候補、B をフォールバックにする。KaTeX 起動時に `auto-render` を有効化すれば、以降 `ui.HTML(...)` で挿した `$$ ... $$` もすべて拾う。

さらに **代入式の整形ルール** を導入:
- `1.184e+04` → `1.184 \times 10^{4}` （`\mathrm{V}` 付き）
- SI 接頭辞は明示: `11840 V → 11.84\,\text{kV}`, `0.0187787 J → 18.78\,\text{mJ}`, `939.029 W → 939\,\text{W}`
- 桁数は **有効 4 桁** を基本
- 時間は μs/ns の自動変換、電流は A/mA、エネルギーは J/mJ/μJ、周波数は MHz/kHz

実装は `format_si()` という小さなヘルパを `report/trace.py` に追加して、`TraceResult.substitution_latex` 生成時にこれを通す。

## 2. タブ別 — 現在の計算式 一覧と、追加する解説テンプレート

以下、**実装ファイル / 式 / 意味 / グラフの見どころ / 結果からの考察** を全物理量について事前定義する。

### 2.1 Upload タブ

メタ情報のみ。計算なし。追加: ファイル読み込み直後に「波形サマリカード」を置き、
- N, Δt, T, fs, Vpp, Ipp を表示
- 「この測定から計算できること」の一覧リンク（タブへのジャンプ）

### 2.2 Waveform タブ

プロットのみ。追加する **読み方ガイド**:
- 「青線が V(t)、赤線が I(t)。電流ピークが電圧ピーク直後に立っていれば、誘導性負荷（ストリーマ形成中の抵抗→コンダクタンス変化）を示唆。」
- 「正負対称なら双極性駆動、片側のみ大きければ偏った整流。」
- 「ベースラインノイズが ±数% を超えるならプローブ接地やシールドを再確認。」

### 2.3 Electrical タブ

#### 2.3.1 ピーク間電圧 Vpp
```
実装: src/oscillo_plasma_calc/signal/peaks.py:detect_vpp
式:    Vpp = Vmax − Vmin
```
- **意味**: 放電に印加される電圧の最大振れ幅。液中プラズマの励起強度を決める第一の外部パラメータ。
- **グラフの見どころ**: Waveform タブの青線の最上点と最下点。上下の絶対値がほぼ等しい ⇔ 双極性駆動、片方が極端に大きい ⇔ 非対称パルス（片側電極優位）。
- **結果からの考察**: 野村研の液中放電帯では Vpp 5〜12 kV が典型。11 kV 超は **Paschen 破壊電圧を明確に超えているストリーマ領域**、5 kV 付近は **プレブレークダウン（部分放電）** の可能性。現データでは PW1.50 が最大 11.84 kV。
- **根拠**: 2013_CAP-13-1050

#### 2.3.2 ピーク間電流 Ipp
```
src/oscillo_plasma_calc/signal/peaks.py:detect_ipp
Ipp = Imax − Imin
```
- **意味**: 放電チャネルに流れる瞬間電流の最大振れ幅。プラズマの瞬時導電率に比例。
- **グラフの見どころ**: Waveform の赤線。Vpp に対する比 Ipp/Vpp ≈ 0.005〜0.01 A/V が液中ストリーマの典型インピーダンス帯。
- **考察**: PW1.50 で 56 A、PW1.00 で 32 A。パルス幅が長いほど必ずしも電流が増えるわけでは無い → **電極接触抵抗 / 気泡形成タイミング** の効果が出ている可能性。

#### 2.3.3 立ち上がり時間 t_r (10→90%)
```
signal/peaks.py:rise_time
t_r = t(x=0.9 xmax) − t(x=0.1 xmax)
```
- **意味**: 電圧（または電流）の立ち上がり速度を時間で表した指標。ストリーマ形成の高速性を示す。
- **グラフの見どころ**: Waveform で立ち上がり部分を拡大。数ナノ秒レベルなら真性ナノ秒パルス、μs レベルならパルス整形回路由来。
- **考察**: 現データ 2.4 μs は測定窓内の「最大値到達時間」を拾っており、生立ち上がりエッジは Δt=2 ns 付近で発生している可能性が高い。→ 改善案: 検出窓を「最初のピーク周辺 ±50 ns」に絞る UI オプションを追加する。
- **根拠**: 2013_CAP-13-1050

#### 2.3.4 ピーク電圧スルーレート dV/dt, 電流スルーレート dI/dt
```
signal/peaks.py:slew_rate
|dV/dt|_max = max_k |(V_{k+1} − V_{k−1}) / (2 Δt)|
```
- **意味**: 電圧（電流）の瞬時上昇速度の最大値。放電チャネルに電子がどれだけ速く注入されるかの指標。
- **グラフの見どころ**: Waveform の立ち上がりエッジの傾き。スルーレートが大きい ⇔ 短時間で大量のエネルギー注入 ⇔ 高電界短時間ストリーマ。
- **考察**: 3.8×10¹¹ V/s（=380 V/ns）は液中ストリーマの 「高速電場立ち上がり」領域。CO₂ 電離断面積ピーク（電子エネルギー 10〜12 eV）への遷移が短時間で起こる。
- **根拠**: 2013_CAP-13-1050

#### 2.3.5 瞬時電力 P(t)
```
electrical/instant_power.py:instantaneous_power
P(t) = V(t) · I(t)
```
- **意味**: 各時刻におけるプラズマへの瞬間的なエネルギー注入率。正なら吸収、負なら放出（浮遊容量の放電）。
- **グラフの見どころ**: Electrical タブの P(t) プロット。ピークが鋭く立ち、幅が数十 ns 程度 ⇔ パルス放電が成立。負方向の振れが大きい ⇔ 回路リンギング / 浮遊 LC 共振。
- **考察**: PW1.50 でピーク 135.6 kW。同じ Vpp·Ipp/4 の理論上限（11840·56/4=166 kW）に迫る → **V と I がほぼ同位相** = 抵抗性負荷に近い。液中ストリーマ電流は主に Joule 加熱として消費される。
- **根拠**: 2006_JJAP-45-8864, 2011_PSST-20-034016

#### 2.3.6 吸収エネルギー E
```
electrical/energy_integral.py:absorbed_energy
E = ∫_0^T V(t) I(t) dt   （合成台形則）
```
- **意味**: 観測窓全体でプラズマに入ったエネルギーの総量。
- **グラフの見どころ**: 累積エネルギー E(t) を副プロットで表示すると、どのパルスにエネルギーが集中しているか一目でわかる。
- **考察**: PW1.50 で 18.78 mJ / 20 μs。これを 10 kHz PRF で動かすと 188 W の平均消費。**反応器冷却が必要なラインを超える場合があり、水冷系の導入判断材料**になる。
- **根拠**: 2008_APEX-1-046002

#### 2.3.7 平均電力 P̄
```
electrical/energy_integral.py:mean_power
P̄ = E / T
```
- **意味**: 観測窓全体での時間平均電力。観測窓の長さに依存するため、**「何を 1 周期とみなすか」** を UI 側で明示すべき。
- **グラフの見どころ**: 現データでは 20 μs 窓内に複数パルスが含まれている（ユーザ提示画像でも 3〜4 発）→ 1 パルス当たりのエネルギーを別カウントで並べる必要がある。
- **考察**: P̄ = 939 W (PW1.50) は **20 μs 観測窓平均**。これを単発パルス電力と誤認しない UI ラベル改善が必要。
- **根拠**: 2008_APEX-1-046002

#### 2.3.8 実効値 Vrms, Irms
```
electrical/rms.py
Vrms = sqrt( (1/T) ∫ V(t)^2 dt )
Irms = sqrt( (1/T) ∫ I(t)^2 dt )
```
- **意味**: 等価な直流電力を与える電圧・電流。RF 系での「入力電力」評価に使う。
- **グラフの見どころ**: Vrms が Vpp/(2√2) ≈ 0.354·Vpp に近ければ **正弦波形**、はずれれば **パルス形状**（現データは後者）。Vrms/Vpp 比が 0.1 以下 ⇔ デューティ比の低いパルス列。
- **考察**: PW1.50: Vrms=1239 V, Vpp=11840 V → Vrms/Vpp=0.105 → デューティ率 約 1.1% のパルス列と整合。

#### 2.3.9 Lissajous（V–q）平均電力（Manley 法）
```
electrical/lissajous.py:lissajous_power
q(t) = ∫ I(t) dt,   Area = |∮ V dq|,   P̄_Liss = f · Area
```
- **意味**: 電荷 q を横軸にした V–q 平面上の閉曲線面積が **1 周期あたりの投入エネルギー**。DBD / 誘電体結合系の標準測定法。
- **グラフの見どころ**: Lissajous プロットが閉じた平行四辺形 ⇔ 典型的 DBD。オープンループなら抵抗性優位、ループ幅が広いほど 1 周期消費が大きい。
- **考察**: 現実装は「観測窓＝1 周期」と仮定して f を計算しているので、**実パルス繰返周波数 PRF を UI で指定しないと結果が窓長依存**。入力欄 `prf_hz` を Electrical タブ先頭に移動し、UI で強調。
- **根拠**: Manley 1943, 2013_CAP-13-1050

#### 2.3.10 瞬時インピーダンス Z(t)
```
electrical/impedance.py:instant_impedance
Z(t) = V(t) / I(t)   （|I|<閾値 は NaN）
```
- **意味**: 放電チャネルの時間発展する抵抗率。ストリーマ形成前後で激変する。
- **グラフの見どころ**: 新規サブプロット（追加予定）。立ち上がり前=数 kΩ 〜 MΩ（絶縁）、ストリーマ形成後= 数十〜数百 Ω（導電性チャネル）。
- **考察**: Z(t) が 2 桁以上急落する瞬間が **ブレークダウン時刻** に相当。CO₂ 電離反応開始のトリガ時刻として化学モデルに結合可能。

#### 2.3.11 FFT 支配周波数
```
signal/fft.py:dominant_frequency (via rFFT + Hanning)
```
- **意味**: 電圧波形のスペクトル最大点。駆動 RF 周波数 or パルス繰返しの主成分を拾う。
- **グラフの見どころ**: FFT タブの対数軸スペクトル。27 MHz (RF), 915 MHz, 2.45 GHz (MW) のラインが見えれば駆動源と整合。高調波 (2f, 3f) の強度比で非線形性を評価。
- **考察**: 現データ PW1.50 の支配周波数は〜数百 kHz〜MHz 帯。パルス放電の「繰返し×パルス内リンギング」の混合成分。
- **根拠**: 2008_APEX-1-046002

### 2.4 FFT タブ

上記 FFT の式のみ。追加すべき解説:
- 「低周波数側のピーク → パルス繰返し周波数 PRF」
- 「中間周波数のコム → パルス内 LC リンギング周波数（プローブや出力段のインピーダンス特性）」
- 「ナイキスト (250 MHz) 直下の広がり → 高周波ノイズ、アンチエリアシング疑問」

### 2.5 Plasma タブ

#### 2.5.1 Boltzmann 2 本線法 → Te
```
plasma/boltzmann.py
I_ij / I_kl = (g_i A_ij ν_ij) / (g_k A_kl ν_kl) · exp[−(E_i − E_k)/(kB Te)]
```
- **意味**: 発光分光で 2 本の輝線強度比を測り、そこから電子温度 Te を逆算する。液中プラズマ診断の王道。
- **グラフの見どころ**: 発光スペクトラムで H_α (656.3 nm), H_β (486.1 nm) 等のピーク強度を読み取る。
- **考察**: Te = 1〜2 eV（10⁴〜2·10⁴ K）が液中大気圧プラズマの典型。Te > 3 eV ⇔ 電子エネルギーが CO₂ 電離断面積ピーク（10〜12 eV）に届く → 反応器が還元反応モードに入る判定。
- **根拠**: 2006_JJAP-45-8864, 2009_JAP-106-113302

#### 2.5.2 Stark 広がり → n_e
```
plasma/stark.py
n_e = α(Te) · (Δλ_{1/2})^{3/2}
```
- **意味**: H_α 線の半値全幅 (FWHM) から電子密度を逆算。プラズマ密度診断。
- **グラフの見どころ**: 高分解能分光器で線形状を取り、Lorentz フィッティング。FWHM が 0.1 nm → ne 〜 3×10²¹ m⁻³、1 nm → ne 〜 10²³ m⁻³。
- **考察**: 液中プラズマでは ne = 10²²〜10²⁴ m⁻³ が典型。Debye 長 ≪ ギャップ長ならバルクプラズマ成立。
- **根拠**: 2009_POP-16-033503

#### 2.5.3 Debye 長 λ_D
```
plasma/debye.py:debye_length
λ_D = sqrt( ε0 kB Te / (ne e²) )
```
- **意味**: プラズマの電荷遮蔽距離。λ_D ≪ 系サイズ ⇔ バルクプラズマとして扱える。
- **グラフの見どころ**: 直接のグラフ量ではないが、Plasma タブで ne と Te を入れたとき自動表示。
- **考察**: Te=1 eV, ne=10²² m⁻³ → λ_D ≈ 74 nm。電極ギャップ 1 mm ≫ λ_D なので **準中性プラズマ成立**。

#### 2.5.4 プラズマ周波数 f_p
```
plasma/debye.py:plasma_frequency
f_p = (1/2π) sqrt( ne e² / (me ε0) )
```
- **意味**: 電子プラズマの固有振動数。RF 周波数 f_RF との大小で電磁波の吸収／透過が決まる。
- **考察**: ne=10²² → f_p ≈ 900 GHz ≫ マイクロ波 2.45 GHz → **MW は十分吸収され、プラズマ内部まで侵入できない（スキン効果）**。この境界が反応器形状設計の要。
- **根拠**: 2008_APEX-1-046002

#### 2.5.5 Ohmic 加熱密度
```
plasma/ohmic.py:ohmic_heating_density
p_ohm = σ E²
```
- **意味**: 導電性液体／プラズマ中の局所 Joule 加熱の体積密度。
- **考察**: σ = 0.01 S/m（純水レベル）× E = 10⁶ V/m → p_ohm = 10¹⁰ W/m³ = 10⁴ W/cm³。数 mm³ 領域で kW 級加熱 → 気泡形成とストリーマ伝搬の駆動力。
- **根拠**: 2011_PSST-20-034016

#### 2.5.6 Paschen 破壊電圧
```
plasma/paschen.py:paschen_breakdown_voltage
V_b = B·p·d / [ ln(A·p·d) − ln ln(1 + 1/γ) ]
```
- **意味**: 気体ギャップでの放電開始電圧。pd 積に対して最小値を持つ（Paschen 曲線）。
- **グラフの見どころ**: Paschen 曲線（p·d vs V_b）をプロットし、現在の測定条件を赤点でマークする追加プロットを提案。
- **考察**: 1 atm × 1 mm → V_b ≈ 3 kV。実測 Vpp=11.8 kV は余裕で超えており、ブレークダウン確定。ただし **液中ではパラメータ A, B が気相と異なる** ため、今の値は「気相 Paschen との比較参照」に留める UI 注記を入れる。
- **根拠**: 2013_CAP-13-1050

### 2.6 Chemistry タブ

#### 2.6.1 G 値
```
chemistry/g_value.py
G = N_prod / (E_abs / 100 eV)   [molecules / 100 eV]
```
- **意味**: 吸収エネルギー 100 eV あたりに生成する分子数。放射線化学由来の普遍指標。
- **考察**: 液中プラズマ H₂ 生成で G=1〜10 が典型。G<0.5 ならエネルギー効率悪化、G>10 は触媒効果疑い。
- **根拠**: 2012_IJHE-37-16000, 2020_JJIE_99-104

#### 2.6.2 化学効率 η
```
chemistry/efficiency.py:chemical_efficiency
η = (ΔH · n_prod) / E_plasma   [-]
```
- **意味**: 投入電力のうち化学エネルギーに変換された割合。
- **考察**: 液中 CO₂ 還元 → メタノールの文献値で η=5〜15%。20% 超なら触媒協調・超音波支援などの 「+α 効果」が効いている。
- **根拠**: 2017_JEPE-10-335

#### 2.6.3 選択性 X_k
```
chemistry/selectivity.py
X_k = n_k / Σ_j n_j
```
- **意味**: 全生成物中での目的生成物のモル分率。
- **考察**: CH₃OH 選択性 >50% が油合成の実用目標。H₂ 共生成が多いと直接油化ラインには不利、Fischer-Tropsch 段への燃料として再利用する道が出てくる。
- **根拠**: 2019_IJHE_44-23912

## 3. 実装アーキテクチャ

### 3.1 新ファイル

```
src/oscillo_plasma_calc/docs/
  __init__.py
  explanations.py       # Explanation dataclass と全物理量の定義辞書
  interpretations.py    # 典型値・オーダー判定（「kV クラス ⇒ ストリーマ領域」等のロジック）
  ui_format.py          # format_si(), format_latex_value() — 生 LaTeX を人に優しい表記に
```

### 3.2 データ構造

```python
@dataclass
class Explanation:
    key: str                  # "vpp", "instant_power" ...
    display_name: str         # "ピーク間電圧 Vpp"
    formula_latex: str        # 整形済み (人が読みやすい形)
    meaning: str              # 2〜4 行の平易解説
    graph_hint: str           # どのタブのどのプロットで何を見るか
    interpretation_rules: list[InterpretationRule]   # value に応じて動的に文章生成
    sources: list[str]
```

```python
@dataclass
class InterpretationRule:
    predicate: Callable[[float], bool]   # e.g. lambda v: v > 1e4
    text: str                             # "11 kV 超はストリーマ領域確定..."
```

TraceResult に `explanation: Explanation | None` を追加。`electrical/`, `plasma/`, `chemistry/` の各 compute 関数は `explanations.py` から対応する `Explanation` を取って attach するだけにする。**計算ロジック自体は変えない**。

### 3.3 UI 変更

- `trace_to_html` を **`render_explanation_card(tr)`** に置き換え
- 4 ブロック構造のアコーディオン表示（Bootstrap の `details/summary` でよい）
- MathJax → KaTeX に置換。`ui.include_js(KATEX_CDN)` + `ui.include_css(KATEX_CSS)` + `renderMathInElement` を読み込み時に一度呼ぶ
- `reactive.effect` で UI を差し替えた後に `ui.tags.script("renderMathInElement(document.body);")` を最後に挿入 → DOM 更新後も再レンダ

### 3.4 数値整形 `format_si(value, unit)`

```python
def format_si(value: float, unit: str) -> str:
    # ex.  (11840, "V")   → "11.84 kV"
    #      (0.0187787, "J") → "18.78 mJ"
    #      (2.43e-6, "s") → "2.43 μs"
    #      (3.8e11, "V/s") → "3.80×10^{11} V/s"   ← 接頭辞なし
    ...
```

これで代入済み式が `$$ V_{pp} = 11.84\,\text{kV} $$` のように可読化される。

### 3.5 後方互換

- 既存 `TraceResult` のフィールドは温存し、`explanation` は Optional で追加
- CLI (`run_analysis.py`) の Markdown 出力は新 4 ブロックを自動追記
- pytest は現状維持（explanation 付加しても既存テストは通る）
